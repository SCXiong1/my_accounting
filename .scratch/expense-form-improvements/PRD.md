## Problem Statement

用户在记账和筛选支出记录时，分类和标签是独立选择的，没有联动关系。这导致：
1. 选择分类后，标签列表显示所有标签，用户需要从大量无关标签中查找
2. 选择标签后，分类列表不联动，无法快速定位相关分类
3. 分类管理入口缺失，用户无法新增自定义分类
4. 记账表单字段顺序不符合直觉，金额在最前面
5. 支出记录页缺少时间范围筛选功能
6. 回收站只支持单条恢复，缺少批量清理功能

## Solution

1. 基于历史记录实现分类和标签的隐式联动
2. 在"我的"页面添加分类管理入口
3. 调整记账表单字段顺序
4. 支出记录页添加时间范围筛选
5. 回收站支持批量选择删除

## User Stories

1. As a 用户, I want 选择分类后标签列表只显示该分类下用过的标签, so that 快速找到相关标签
2. As a 用户, I want 选择标签后分类列表只显示该标签用过的分类, so that 快速定位相关分类
3. As a 用户, I want 清空分类时自动清空标签选择, so that 保持筛选条件的一致性
4. As a 用户, I want 切换分类时自动清空标签选择, so that 避免选择不属于新分类的标签
5. As a 用户, I want 标签管理页创建的标签作为全局标签, so that 可以在任何分类下使用
6. As a 用户, I want 选了分类后只显示该分类下的标签（不含全局标签）, so that 标签列表更精准
7. As a 用户, I want 选择全局标签后分类列表显示所有分类, so that 不限制分类选择
8. As a 用户, I want 在"我的"页面找到分类管理入口, so that 可以新增、编辑、删除自定义分类
9. As a 用户, I want 记账时字段顺序为分类→标签→日期→备注→金额→提交, so that 先选分类标签再填金额，符合直觉
10. As a 用户, I want 在支出记录页按时间范围筛选, so that 快速查看特定时间段的支出
11. As a 用户, I want 时间范围、分类、标签三个筛选条件组合使用, so that 精确筛选支出记录
12. As a 用户, I want 在回收站批量选择记录并永久删除, so that 一次性清理不需要的记录
13. As a 用户, I want 批量删除前弹出确认对话框显示删除数量, so that 防止误操作
14. As a 用户, I want 我的分类和标签数据与其他用户隔离, so that 不会影响其他用户的数据
15. As a 用户, I want 新增的分类自动出现在记账表单和筛选器中, so that 立即可以使用
16. As a 用户, I want 删除分类时如果有支出记录则无法删除, so that 保护数据完整性
17. As a 用户, I want 现有的分类和标签数据无需迁移即可支持联动, so that 平滑过渡到新功能
18. As a 用户, I want 在记账时快速新增标签并自动关联当前分类, so that 提高记账效率
19. As a 用户, I want 支出记录页的时间筛选器复用统计分析页面的交互方式, so that 保持一致的用户体验
20. As a 用户, I want 回收站的批量删除操作不可撤销, so that 明确操作风险

## Implementation Decisions

### 1. 分类和标签联动机制

**决策：基于历史记录推断（隐式关联）**

不修改数据库结构，通过查询 `expense` 和 `expense_tag_index` 表的关联关系来推断分类和标签的对应关系。

**查询逻辑**：
- 查询某分类下的标签：通过 `expense.category_id` 和 `expense_tag_index.tag_id` 联合查询
- 查询某标签下的分类：通过 `expense_tag_index.tag_id` 和 `expense.category_id` 联合查询

**联动规则**：
- 选分类 → 标签列表只显示该分类下用过的标签（不含全局标签）
- 选标签 → 分类列表只显示该标签用过的分类（全局标签则显示所有分类）
- 清空分类 → 自动清空标签选择
- 切换分类 → 自动清空标签选择
- 筛选逻辑：分类 AND 标签（同时满足）

**全局标签处理**：
- 标签管理页创建的标签作为全局标签（未关联任何分类）
- 未选分类时，标签列表显示所有标签（全局标签 + 各分类下用过的标签）
- 选了分类后，标签列表只显示该分类下用过的标签（不含全局标签）
- 选择全局标签后，分类列表显示所有分类

**现有数据处理**：
- 无需数据迁移，基于历史记录的查询方式自动支持已有数据

### 2. 分类管理入口

**决策：在"我的"页面添加"分类管理"入口**

复用已有的 `CategoryManagePage`，在 `ProfilePage` 中添加导航链接。

**已有功能**：
- CategoryManagePage 已实现完整的 CRUD 功能
- 新增分类自动分配排序序号，关联当前用户
- 修改分类只能改名称、图标、颜色
- 删除分类时检查是否有支出记录关联，有则无法删除
- 支持拖拽调整排序

**用户隔离**：
- 每个分类都有 `uid` 字段，用户增加分类只影响自己
- 注册时创建的 8 个预设分类也是用户级别的

### 3. 记账表单字段顺序

**决策：调整为分类→标签→日期→备注→金额→提交**

修改 `ExpenseFormPage.vue` 中的字段顺序，将 `AmountField` 移到最后。

### 4. 支出记录页时间筛选

**决策：顶部时间选择器 + 现有分类、标签筛选**

复用 `usePeriodFilter` composable，在支出记录页顶部添加时间选择器。

**筛选条件**：
- 时间范围：本月、近3月、近6月、今年、自定义
- 分类：单选
- 标签：单选
- 三个条件独立显示，AND 逻辑组合

**交互方式**：
- 顶部时间选择器按钮组（类似统计分析页面）
- 保持现有的分类、标签筛选入口
- 三个筛选条件可以同时生效

### 5. 回收站批量删除

**决策：批量选择删除 + 二次确认**

在回收站页面添加批量选择功能，支持勾选多条记录后批量永久删除。

**交互流程**：
- 页面添加"批量删除"按钮，进入选择模式
- 选择模式下每条记录显示复选框
- 选择记录后点击"删除"按钮
- 弹出确认对话框，显示将删除的记录数量
- 确认后批量永久删除

**安全机制**：
- 永久删除不可逆，必须二次确认
- 确认对话框显示删除数量，提醒用户风险

## Testing Decisions

### 测试策略

**测试层次**：
- 单元测试：测试联动查询逻辑、筛选条件组合
- 集成测试：测试 API 端点的筛选功能
- E2E 测试：测试完整的用户交互流程

**测试场景**：
1. 分类和标签联动
   - 选分类后标签列表正确过滤
   - 选标签后分类列表正确过滤
   - 全局标签的显示逻辑
   - 清空/切换分类时标签自动清空
2. 分类管理
   - 新增分类后出现在记账表单和筛选器
   - 删除分类时有支出记录的处理
3. 记账表单
   - 字段顺序正确
   - 切换分类时标签清空
4. 时间筛选
   - 预设时间段筛选正确
   - 自定义时间段筛选正确
   - 与分类、标签筛选组合使用
5. 回收站批量删除
   - 批量选择功能正常
   - 确认对话框显示正确
   - 批量删除后列表更新

**测试数据**：
- 使用现有测试数据，无需额外准备
- 联动功能基于历史记录，测试数据自然支持

## Out of Scope

1. **数据库结构修改**：不添加分类和标签的直接关联字段
2. **标签管理页显示关联分类**：不在标签管理页显示标签和分类的关联关系
3. **自动清理回收站**：不实现超过一定天数自动永久删除的功能
4. **批量恢复功能**：本次只实现批量删除，不实现批量恢复
5. **分类模板功能**：不实现分类方案的保存和导入
6. **标签批量管理**：不在标签管理页添加批量删除功能

## Further Notes

### 实现优先级

建议按以下顺序实现：
1. 分类管理入口（最简单，复用已有功能）
2. 记账表单字段顺序调整（简单 UI 调整）
3. 分类和标签联动（核心功能，需要前后端配合）
4. 支出记录页时间筛选（复用已有 composable）
5. 回收站批量删除（独立功能，不影响其他）

### 性能考虑

- 分类和标签联动查询基于历史记录，如果用户数据量大，可能需要优化查询性能
- 可以考虑缓存分类和标签的关联关系，减少数据库查询
- 时间筛选使用现有的 `usePeriodFilter`，性能已有保障

### 用户体验

- 联动功能应平滑过渡，不影响现有使用习惯
- 新增分类后应立即出现在相关选择器中
- 筛选条件应清晰显示当前状态
- 批量删除操作应有明确的反馈
